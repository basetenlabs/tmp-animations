Hierarchical caching for multimodal responses

Click through in order to watch one image warm the hierarchy. It starts cold on plane A, gets reused across planes, and settles into the local cache on plane B. Pre-warm is a separate upload-time optimization.

routing path cache miss path

The story. A new image lands on plane A — everything is cold, so the request walks the full hierarchy and pays the L4 encode cost. The same image later arrives on plane B (dynamic routing): its local cache is empty, but central has it from plane A's compute — central hit. A follow-up turn on plane B now finds it locally. Pre-warm is the orthogonal trick: on upload, the client fires a warm request to the global entrypoint so central cache is populated before the prompt is even sent.
